# p05 — Weekly completion-rate chart (last 8 weeks) on /stats

## 1) Work summary

- **`lib/completion.ts`** — pure `weeklyCompletionRates(dates, habitCount, today, weeks)`: buckets the multiset of check-in keys into Sunday-start calendar weeks (matching the calendar UI), for the `weeks` weeks ending with today's week. Rate = check-ins ÷ (habit count × elapsed days); the current week's denominator only counts days up to today, future-dated rows are ignored, and an empty denominator yields 0.
- **`lib/date.ts`** — two new helpers: `startOfWeek` (Sunday-based) and `shortDateLabel` ("Jun 7"), keeping all date logic centralized per the standing rule.
- **`components/features/WeeklyCompletionChart.tsx`** — Recharts bar chart (`'use client'` with the required justification comment): 8 columns, y-axis 0–100% with 0/50/100 ticks, horizontal-only recessive grid, rounded bar tops anchored to the baseline, hover tooltip showing week range + percent + raw counts, and an sr-only table mirroring the data (also used by the e2e as the machine-readable source of truth). All chart colors come from new CSS tokens in `globals.css` (`--chart-accent/grid/muted-text`, dark-aware) — no hex values in components.
- **`app/stats/page.tsx`** — computes the 8-week series from the check-in dates it already fetches for streaks (no extra query) and renders the chart card above the per-habit stat cards.
- Dataviz procedure followed: form chosen (magnitude over discrete weeks → column chart, single series so no legend), the accent color was **validated with the skill's palette script, not eyeballed** — `#0284c7` passes all six checks on both the light (#fff) and dark (#0a0a0a) surfaces, so one token serves both modes — and the rendered result was screenshot-reviewed in light and dark.

## 3) Why Recharts

- **Fits the stack**: first-class React composition (declarative `<BarChart><Bar/>…`) that slots into a Next.js client component with zero config; v3.9 supports React 19 / Next 16 out of the box (verified by this build).
- **Right altitude for the job**: an 8-bar chart needs axes, a tooltip, and responsive sizing — Recharts provides exactly these as primitives without the low-level assembly cost of d3/visx or the canvas indirection of Chart.js (SVG output also made the e2e and accessibility assertions straightforward).
- **Ecosystem weight**: the most widely used React charting library — maintained, documented, easy for any future contributor.
- **Cost containment**: the dependency's bundle impact lands only on the `/stats` route chunk (App Router code-splitting), and the data math deliberately lives outside it in pure, server-tested `lib/` code, so the library is a swappable rendering detail.
- Alternatives considered: Chart.js/react-chartjs-2 (canvas — weaker SSR/test story, wrapper indirection), visx/d3 (more control than an 8-bar chart justifies), Tremor (imposes its own design system on top of Recharts anyway), hand-rolled SVG (the task asked for a library, and a library buys tooltip/scale correctness).
