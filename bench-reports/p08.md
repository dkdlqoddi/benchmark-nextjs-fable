## 4) Key decisions and reasons

- **Audience and depth**: wrote for a developer/agent joining the project — every claim traced to code read during analysis (no speculation), with file paths throughout so statements are checkable. Ended with an "invariants checklist" distilling what must not be broken, since that is what a future change most needs.
- **Verified framework behavior against bundled docs, not memory** (per `AGENTS.md`): the caching section states that Cache Components is off and `force-dynamic` is the deliberate consequence — the docs skim confirmed the opt-in `cacheComponents: true` model, so the doc also notes those exports are the revisit-inventory if it is ever enabled.
- **Documented the "why" behind non-obvious modeling**: `CheckIn.date` as a TZ-proof string key, the P2002-tolerant toggle, the explicit delete transaction (SQLite `PRAGMA foreign_keys` is driver-dependent), and server-only validation (native form attributes intentionally omitted) — these are the traps a newcomer would otherwise re-litigate.
- **Kept `README.md` untouched**: README stays the quick-start; ARCHITECTURE.md is the deep structural reference and links to README/CLAUDE.md rather than duplicating them.
- **Ran the full quality suite despite a docs-only change**: cheap, and it makes the "repo left green" claim evidence-backed; Prettier does check root markdown (only `bench-reports/` is ignored), so formatting the new doc was required, not optional.
